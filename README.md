# WP Announcements

Универсальный плагин WordPress/WooCommerce для работы с **анонсами** — товарами, которые пока нельзя купить, но можно оформить предзаказ (оставить email) и получить уведомление, когда товар появится в обычном каталоге.

Работает с любой тематикой: книги, техника, одежда, билеты, услуги — что угодно. Определение «анонс» — по выбранным категориям product_cat.

## Возможности

- Выбор одной или нескольких `product_cat` как «анонсных».
- Товары из анонсных категорий автоматически:
  - становятся непокупаемыми (`is_purchasable = false`);
  - получают кнопку **«Предзаказ»** вместо «В корзину» — и в каталоге, и в карточке товара (работает в любых темах через 3 фильтра).
- Клик по «Предзаказ» → модалка с полями: имя (необяз.), email (обязательно) + honeypot.
- Подписки пишутся в таблицу `wp_wpan_subs`, в админке — список с пагинацией и **CSV-экспортом**.
- Авто-рассылка через **WP-Cron** при переходе товара из анонсной категории в обычную. Работает и при ручной правке, и при импорте из МойСклад.
- Шорткоды:
  - `[announcements_grid limit="8" columns="4" title="Скоро в продаже"]` — сетка для главной.
  - `[announcements_page per_page="24" columns="4"]` — страница со всеми анонсами и пагинацией.
- Rate-limit по IP (настраивается) + honeypot-поле против ботов.
- Настройки карточки: бейдж, показ меток/цены, цвета (CSS-переменные).
- Опциональная очистка БД и опций при удалении плагина.

## Установка

1. Скопировать папку `wp-announcements/` в `wp-content/plugins/`.
2. Активировать — создастся таблица `wp_wpan_subs` и запланируется WP-Cron.
3. **Анонсы → Настройки** — выбрать категории, настроить тексты/цвета/шаблоны писем.

## Как это работает

- При любом изменении категорий товара (`set_object_terms`) плагин читает **актуальный** набор категорий через `wp_get_object_terms` и сравнивает с анонсными.
- Если товар был анонсом, а стал обычным — товар помечается мета-флагом `_wpan_notify_pending`.
- WP-Cron каждый час (и через 30 сек после триггера) обходит очередь и рассылает письма через `wp_mail`. Это защищает SMTP при массовых изменениях категорий.
- После успешной отправки в таблице подписок ставится `notified_at`. Админу уходит отчёт с количеством получателей.

## Интеграция с МойСклад (wooms)

Никаких настроек не требуется: перенос товара из анонсной папки в обычную в МС меняет `product_cat` стандартным способом, хук `set_object_terms` отрабатывает автоматически.

## Переменные в шаблонах писем

- `{product}` — название товара
- `{product_url}` — ссылка на товар
- `{site_name}` — название сайта
- `{count}` — количество уведомлённых (только в админском письме «рассылка выполнена»)

## Хуки для разработчиков

- `_wpan_is_announcement` (post_meta) — `'1'`/`'0'`, актуальный статус товара.
- `_wpan_notify_pending` (post_meta) — `'1'` пока ожидает обработки в Cron.
- `wpan_send_notifications` (cron hook) — можно дёрнуть вручную через `do_action('wpan_send_notifications')`.
- Таблица `{prefix}wpan_subs`: `id, product_id, email, name, created_at, notified_at, ip`. UNIQUE на `(product_id, email)`.
